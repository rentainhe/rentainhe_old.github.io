Torch API used in Matcher

  • tensor.numel()

返回这个张量中总的元素个数, 用法和torch.numel()一样

  • tensor.new_full(): document

torch中有一部分内置的类似用法, 例如: torch.ones(size), torch.zeros(size), 可以创建一个size大小的tensor, 并且其中全部为1或者0, 但是如果想得到一个全为一个指定的值的tensor的话, 就可以用到tensor.new_full方法, 注意这里是tensor.new_full, 所以需要依赖于某个tensor来使用, 默认返回与这个tensor相同device和dtype的一个新的tensor

  • tensor.max()

沿着索引返回最大值, 以及其对应的index

dim=0表示沿着0这个维度去进行比较, 也就是从上到下进行比较, 返回每个列的最大值, 以及其对应的index

Matcher

  • Matcher的初始化: 传入thresholds, labels和allow_low_quality_matches三个参数
    • thresholds: 一个list, 包含不同级别的threshold, 用来区分不同level的prediction, RetinaNet中默认设置为[0.4, 0.5]
    • labels: 不同level的thresholds对应的label, 比如{-1, 0, 1}分别对应{ignore, negative class, positive class}

如果指定了thresholds为[0.3, 0.5], labels为[0, -1, 1]的话, 意味着iou < 0.3的情况下, 会被分配0标签, 作为false negatives, 0.3 <= iou < 0.5会被分配-1标签, 表示被ignored, 0.5 <= iou会被分配1标签, 作为true positives.
