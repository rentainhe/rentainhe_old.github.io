Logging模块的基本信息

1. 日志等级

logging默认定义了5个日志等级, 从高到低分别为:

DEBUG < INFO < WARNING < ERROR < CRITICAL

在指定了日志的级别之后, 只会记录大于等于这个日志级别的信息, 小于的将被丢弃, 默认的等级是WARNING, 所以只会记录ERROR, CRITICAL级别的日志信息

2. 日志字段与格式

一个日志信息对应的是一个事件, 一个事件通常需要包括以下几个内容:

  • 事件发生的时间
  • 事件发生的位置
  • 事件的严重等级
  • 日志级别, 事件内容等

Notes: 只有日志内容和日志级别是需要明确指定的, 其他字段信息只需要指定是否显示即可

3.5 不同组件之间的关系
  • 日志器 (logger) 需要通过处理器 (handler) 将日志信息输出到目标位置，不同的处理器 (handler) 可以将日志输出到不同的位置。
  • 日志器 (logger) 可以设置多个处理器(handler), 将同一条日志记录输出到不同的位置。
  • 每个处理器(handler)都可以设置自己的过滤器 (filter) 实现日志过滤，从而只保留感兴趣的日志。
  • 每个处理器 (handler) 都可以设置自己的格式器 (formatter) 实现同一条日志以不同的格式输出到不同的地方。
