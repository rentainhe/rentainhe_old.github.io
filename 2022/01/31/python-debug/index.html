记录一下Python中如果报错找不到Module时的解决方案

python的工作路径

  • 获取python进程的工作路径, 进程的工作路径取决于在哪里执行这个文件, 如果是python test.py, 那么工作路径就是./, 如果是python test/test.py, 那么工作路径就是/test
  • 修改工作路径
  • 工作路径主要影响程序里使用的相对路径

python报错找不到Module的解决方案

问题描述

假设我们的目录结构为

假设在debug.py中我们使用了这样的一段代码, 并且我们在project/benchmark/classification/下执行python debug.py

报错ModuleNotFoundError: No module named 'flowvision'

解决方案

  • 首先使用sys.path查看当前执行的代码里，可以找到的module有哪些

我们得到了当前执行的代码里可用的module，可以看到，第一个是'/home/rentianhe/code/OneFlow-Models/vision/projects/benchmark/classification'，表示我们当前执行的路径下，没有vision的路径，所以就无法找到flowvision这个module

  • 手动将vision这个module添加到sys.path中

再次执行，发现成功了，这时候我们已经把vision这个module添加到了sys.path中，所以可以找到vision下的flowvision模块
